MTP (Mapping the Prompt) は、長い自然言語による振る舞い指示の代わりに、構造化されたコントロール（スライダー、グリッド座標、プリセット）で LLM の出力を制御するためのフレームワークです。

モデルのトーンや語り口を言葉で直接指示する代わりに、**座標ベースのコンパイル手順**で制御します。「簡潔に」「深く考えて」といった直接的な指示に頼るのではなく、構造化された空間内の位置を指定します。指定方法は、スライダー値、グリッド座標、名前付きプリセットのいずれでもかまいません。コンパイラはその位置を決定論的に変換し、出力の形を変える制約 XML に落とし込みます。

このページでは、**基本用語**と、MTP の制御を支える二つのレイヤーの簡易仕様をまとめます。この二つのレイヤーは、実際の挙動に即して言えば **グリッド上の位置** と **強度・極性** として整理でき、内部モデルでは **Space** と **Intensity** と呼びます。計算式やゾーン境界の完全な定義はグリッドと座標系で、各ノードの振る舞いの要約はノードリファレンスで扱います。

Space の基準となる 3x3 の色配置は次のとおりです。

| 位置 | 左          | 中央          | 右        |
| ---- | ----------- | ------------- | --------- |
| 上段 | Yellow      | Red           | Magenta   |
| 中段 | Green       | Transparent   | White     |
| 下段 | Cyan        | Blue          | Purple    |

## 基本用語

以下の用語は、MTP のドキュメント全体で繰り返し登場します。これらは、入力から制約の適用までをつなぐパイプラインを構成しています。

```text
  ┌─────────┐     ┌──────────────────┐     ┌───────────────────────────┐     ┌─────────────┐
  │  Input  │ ──→ │     Compiler     │ ──→ │  Constraint extraction    │ ──→ │   Output    │
  └─────────┘     └──────────────────┘     └───────────────────────────┘     └─────────────┘

  プリセット ───── 展開 ─────→ グリッド座標 ────────┐
  スライダー (node:intensity)  ────────────────→ ├→ (軸, 極性, 強度) → ノードファイル → 制約
  グリッド座標 (列:行)  ────────────────────────→ ┘
```


| 用語                           | 意味                                                                                             |
| ---------------------------- | ---------------------------------------------------------------------------------------------- |
| **ノード (Node)**               | 9 つある意味軸のうちの 1 つです（例: Red）。各ノードには Side A の名称（例: Power）と Side B の名称（例: Void）があります。              |
| **軸 (Axis)**                 | ノードの色による識別単位です。Yellow, Red, Magenta, Green, Transparent, White, Cyan, Blue, Purple の 9 種があります。 |
| **Side A / Side B**          | 各軸の両極です。強度が正なら Side A、負なら Side B が有効になります。                                                     |
| **強度 (Intensity)**           | 0〜100 の強度値です。どの制約ティア（Low / Mid / High）が抽出されるかを決めます。                                            |
| **スライダー (Slider)**           | `node:intensity` 形式の入力です。例: `power:70`, `void:80`, `yellow:-30`                                |
| **グリッド (Grid)** | 19×19 グリッド上の `列:行` 形式の入力です。例: `D:16`, `A:1`。コンパイラは位置から軸、極性、強度を計算します。                           |
| **ゾーン (Zone)**               | グリッドを 3×3 に分割したときの 9 つの大きなセルのうちの 1 つです。各ゾーンは 1 つの軸に対応します。                                      |
| **プリセット (Preset)**           | パース前にグリッド座標へ展開される名前付きブレンドです。例: `strategist` は `P:16 P:4` に展開されます。                              |
| **制約 (Constraint)**          | 解決された強度ティアに応じてノードファイルから抽出される、出力の振る舞いを形作る指示群です。                                                 |


どの入力経路も、最終的には同じ内部表現である **(軸, 極性, 強度)** に収束します。たとえば `power:100`、`red:100`、`J:4` のどれを入力しても、コンパイラは同じ 3 要素に解決し、同じ制約を抽出します。

| <div class="dot-sm bg-power" aria-label="red"></div> | Red         | Power  | Void     |

## 二つのレイヤー

MTP は形式上、**Space** と **Intensity** という二つのレイヤーで成り立っています。この概要では、これらを **グリッド上の位置** と **強度・極性** という、より具体的な見方で導入します。**Space** は「グリッド上のどこか」、つまり軸とゾーンを扱います。**Intensity** は「どれくらい強いか」と「どちらの極か」を扱い、グリッド座標からの解決には **Volcano** マッピングとティア抽出が関わります。3×3 レイアウトの設計背景は設計上の背景で扱います。

### レイヤー一覧


| レイヤー          | 制御するもの                   | 主な考え方                                             |
| ------------- | ------------------------ | ------------------------------------------------- |
| **Space**     | *グリッド上の位置*： どの軸・どのゾーンか   | 19×19 グリッド、3×3 マクロゾーン、色相環                         |
| **Intensity** | *強度・極性*： どれくらい強いか、どちらの極か | 0–100 スライダー、Side A / Side B、中心からの距離に応じた強度・極性の対応づけ |


各トークンについて、コンパイラは **(軸, 極性, 強度)** を解決し、ノードファイルから段階的な制約を抽出します。

### Space（グリッド上の位置）

インターフェースでは、**19×19 の線の交点**を基準にした整数座標系を使います。これは現在のデフォルト実装です。盤面は対称な 3×3 レイアウトで **9 つのマクロゾーン** に分かれており、各ゾーンは 1 つの色軸に対応します。この配置の背景は設計上の背景で扱います。

### Intensity（強度・極性）

各ノードには **0〜100** の連続的な強度があります。**極性**によって、Side A（正）か Side B（負 / 反転極）かが決まります。マクロゾーンの色名はスライダー名の別名としても使えます（例: Yellow 軸では `yellow:50` ≡ `open:50`）。グリッド上の位置から強度と極性は、**チェビシェフ距離に基づく径方向のルール**で求められ、その定義はグリッドと座標系にあります。見た目の対応としては、中心が中立、外側の環が Side A のピーク、外周が Side B です。

複数の MTP トークンを並べたときは、コンパイラは**トークン順**に `<constraints>` ブロックを出力します（別レイヤーではありません）。並びの設計上の任意の読み方（軌跡の比喩など）は設計上の背景で、各ノードの要約はノードリファレンスで扱います。

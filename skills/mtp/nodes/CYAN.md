# Cyan Axis (Enter / Drift) Constraints

## Side A: Enter (positive / entry, landing, structure)
Direction: Clear intro design, state the goal, structured onboarding, logical framing.

### Low (intensity 1–30): Slight entry
- **Reasoning/output:** Briefly state context and scope at the start (e.g. "This question is about X, so I will answer from Y.").
- **Structure:** Define the topic in the first one or two sentences so the reader is oriented.

### Mid (intensity 31–70): Clear structure
- **Reasoning/output:** Put an outline or roadmap at the start ("I will cover three points: ①… ②… ③…"). Declare the goal in advance.
- **Structure:** Clear sectioning, numbered steps, explicit assumptions. Let the reader see the big picture before the details.

### High (intensity 71–100): Full framing
- **Reasoning/output:** Define task assumptions, constraints, and success criteria at the start. Structure the answer like a spec; eliminate ambiguity.
- **Structure:** Use a phase structure (requirements → design → implementation → verification). Give each section clear input/output definitions.

---

## Side B: Drift (negative / drift, deviation, chance)
Direction: Allow deviation from the goal, serendipity, creative value of tangents.

### Low (intensity -1–-30): Slight drift
- **Reasoning/output:** Allow a small tangent. Naturally add a "by the way" or "incidentally."
- **Structure:** Allow slight deviation from strict structure; do not reject unplanned turns.

### Mid (intensity -31–-70): Clear deviation
- **Reasoning/output:** Allow the answer to drift from the original question along associative lines. Develop related topics in a free-associative way.
- **Structure:** Loosen logical links between sections; let each paragraph stand as an independent snapshot.

### High (intensity -71–-100): Full drift
- **Reasoning/output:** Abandon a direct answer to the question; output the train of thought triggered by it. A conclusion is optional.
- **Structure:** Stream-of-consciousness. No clear beginning or end; text that drifts.
